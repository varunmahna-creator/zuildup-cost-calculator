## Lessons reinforced today (already in MEMORY.md)

### 🚨 "Deploy SHA matches" ≠ "Verified"
Item 6 round 1: I claimed verified because Vercel SHA matched HEAD and MD5 was consistent. Didn't actually log an activity. Varun caught it → cost the sales team a whole night of manual refresh. Real root cause only found on round 2: backend wasn't writing `next_action_type`/`next_action_notes` to the lead row, so the list was visually unchanged after save.

**Rule:** "Done" = opened live URL, performed actual user action, saw actual user-visible result. For auth-gated APIs: mint JWT, POST real payload to prod, query resulting DB state, AND verify GET endpoint returns new state. Three layers of proof.

### 🚨 Read schema enums BEFORE inventing string values
Item 8 round 2: invented `'Follow-up Call'` as `next_action_type`. `action_type` is a Postgres ENUM. Every POST /activity returned 503 (22P02) for ~60s on rev `00024-hv9`. R2 E2E test caught it immediately.

**Rule:** Before writing ANY value to an enum column, run `SELECT enum_range(NULL::<enum_name>)` and use one of those values.

### 🚨 Enum-mismatch silent timebombs (item 7)
A new sub_status added in a frontend sprint needs to be added in 3 places in the SAME PR:
1. Frontend `SUB_STATUS_BY_TOP` (`web/lib/format.ts`)
2. Backend `TOP_TO_SUBS` (`shared-db/index.js`)
3. DB `chk_status_sub` CHECK constraint

The May 25 sprint did 2/3 for the new Qualified 'Call Done' entry. Was a silent timebomb until 2026-05-28 when a SPOC actually picked it in prod.

### 🚨 FS quirk on this VM
`cd /opt/openclaw/workspace/<deep-dir>` and `ls/cat/Read` on absolute paths often returns ENOENT for files that exist (`find -exec` always works). Workarounds:
- Symlink: `ln -sfn /opt/openclaw/workspace/zuildup/sales-os /tmp/sos` then operate via `/tmp/sos/...`
- `find <dir> -name '<file>' -exec cat {} \;` always works
- `>>` redirection often fails silently — write to `/tmp/foo` then `cat /tmp/foo >> /tmp/sos/target` via symlink
- `gcloud secrets versions access ... | other-cmd` sometimes drops output → redirect to file first: `gcloud secrets ... > /tmp/foo.txt && OTHER=$(cat /tmp/foo.txt)`
